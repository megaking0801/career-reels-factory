# 企劃與策略：AI 虛擬職涯導師 IG 帳號

> 本檔 = 帳號定位策略 + 自動化工具實作計畫的合併版。
> 工具程式碼在同層的 `career-reels-factory/` 資料夾。

---

## 一、為什麼做這個（Context）
打造一個**全新 IG Reels 帳號**（純虛擬品牌、不露真人臉），用 AI 生成「藏鏡人街訪式」職場/求職知識短影音。
終極目的：**內容導流 → 養大帳號 → 開線上課程變現**。

**已確認的關鍵決策**
- 受眾：**先廣泛衝量**，之後依數據再 niche down
- 變現：**純虛擬品牌**，課程之後再找師資/合作或掛品牌賣
- AI 角色：**混合**——虛擬導師講觀念 + 情境模擬演面試/離職短劇
- 平台：**先 IG Reels**，看成效再擴其他平台
- 成本：**先全程 $0**，工具皆可抽換，之後再升級付費全自動

---

## 二、帳號定位（一句話）
> 一個 AI 虛擬的「職場前輩」IP，用邊走邊聊＋情境短劇，戳破職場迷思、講求職與職場生存的真心話。

**角色分工（藏鏡人格式）**
- 鏡頭前（固定 IP，每集同一人）：AI 虛擬「職涯前輩」= 帳號的臉，日後課程要承接的品牌
- 鏡頭後（藏鏡人，只有聲音/字卡）：提問者，代表觀眾的困惑，丟痛點問題
- 混合呈現：平常導師邊走邊答；面試/離職/談薪類切成情境模擬短劇

**調性**：知識型，但有「說真話、反差、戳破迷思」的記憶點 → 易被儲存/分享，也建立「願意付錢學」的信任。

**人設（預設，可在 config.json 改）**：毒舌但中肯的職場學長/姐。

---

## 三、內容支柱（先廣泛，4 條線輪流）
1. **求職硬技能**：履歷、面試、薪資談判、找工作管道、選 offer
2. **職場生存**：跟主管/同事相處、該不該離職、被裁、加薪
3. **迷思破解／真心話**：「大家以為…其實…」
4. **情境模擬短劇**：面試現場、離職對話、談加薪

> 養粉期用這 4 條線當探針，看哪類數據最好 → 那就是 niche down 與**課程主題**的依據。

---

## 四、單集格式（IG Reels 9:16，15–40 秒）
- **0–2s Hook**：藏鏡人痛點問題 / 導師反差金句（最重要）
- **本體**：導師邊走邊答，3 個重點，字幕同步
- **收尾**：記憶金句 + CTA 追蹤
- **固定識別**：同一角色、同一字幕樣式、固定片頭片尾

---

## 五、漏斗（導流 → 課程）
- 免費內容講「what / why」，課程賣「how / 手把手 + 範本 + 個案」
- 養粉期：精選集分類、置頂代表作、限動互動、累積名單
- 變現轉折：用數據找出最紅的內容線 → 第一門課的主題與受眾

## 六、技術棧（先全免費，每棒可抽換）
| 環節 | 方案 | 免費? |
|---|---|---|
| 腳本 LLM | Groq 免費（預設）/ Claude API（繁中更佳） | ✅ |
| 角色圖（虛擬人的臉） | ChatGPT 生圖（DALL·E），一次性固定 | ✅ |
| 口播語音 | edge-tts（免費台灣中文）/ Kling 內建 TTS | ✅ |
| avatar 影片 | **Kling 對口型**（先朋友帳號手動，未來可接 API） | ✅/付費 |
| 字幕辨識 | 本機 faster-whisper | ✅ |
| 字幕燒錄/合成/轉直式 | 本機 FFmpeg | ✅ |
| Web App | Python + FastAPI | ✅ |

> ⚠️ avatar 這棒是整條鏈最難「完全免費」的地方，真正「完全免費＋全自動＋一致中文 avatar」目前幾乎不存在。
> ~~原本規劃用 HeyGen~~ → **已棄用**：HeyGen 免費版每支有浮水印、下載受限，且 2026/02 已取消免費 API。
> 現改用 **Kling 對口型**（給「角色圖＋口播語音」讓虛擬人開口講話，支援中文、可鎖定同一張臉、還能做走動運鏡）；先用手動，架構預留接 Kling API 升級全自動。

## 七、工具運作流程（Kling 對口型 avatar）
0. **（一次性）用 ChatGPT 生一張固定角色圖** = 虛擬前輩的臉，之後每支都用同一張
1. 本機網頁選**內容線/主題**（皆可留空 → AI 自動出題）+ 調整方向
2. App 自動產**腳本**（給口播逐字稿）；口播語音可用 App 端 edge-tts 或 Kling 內建 TTS
3. 在 **Kling** 用「角色圖 + 口播語音」做對口型影片、下載 mp4
4. mp4 上傳回 App
5. App 自動：Whisper 轉字幕 → FFmpeg 燒字幕 + 疊開場問題字卡 + 轉 9:16
6. 成果畫廊預覽下載 → 手動發 IG

> 雙聲處理：導師 = avatar 口播（Kling）；藏鏡人問題 = 開場字卡疊在畫面（App 合成時加，不需第二配音）。

## 九、驗證/Pilot 待辦
> ⚠️ 前置已實測修正：ffmpeg 要用含字幕功能的版本、Python 要用 3.12（詳見 README 安裝段）。
1. `brew install homebrew-ffmpeg/ffmpeg/ffmpeg`（含 libass/libfreetype；一般的 `brew install ffmpeg` 不夠）
2. `brew install python@3.12`（3.13/3.14 太新，faster-whisper 裝不起來）
3. 用 3.12 建 venv：`"$(brew --prefix python@3.12)/bin/python3.12" -m venv .venv && source .venv/bin/activate`
4. `pip install -r requirements.txt`（含 faster-whisper）
5. 填免費 `GROQ_API_KEY` 到 `.env`
6. `uvicorn app.main:app` → http://localhost:8000
7. （一次性）用 ChatGPT 生一張固定角色圖
8. 選/留空主題 → 確認腳本生成 → 拿口播語音＋角色圖到 Kling 做對口型 mp4 → 上傳 → 確認成品有字幕、9:16、開場字卡
9. 發第一支 IG Reels，依數據微調字幕樣式與內容線

> 環境現況（2026-06-29 已建好）：`.venv` 已用 3.12 建好、套件已裝、含字幕 ffmpeg 已裝、`.env` 已填 GROQ 金鑰。下次直接從步驟 6 啟動即可。
