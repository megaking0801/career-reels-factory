# 對話記錄：從發想到做出自動化工具

> 這份是與 Claude Code 的完整討論記錄（重點整理版），記錄這個專案怎麼從一個模糊想法，一步步收斂成可執行的帳號企劃 + 自動化工具。日期：2026-06-29。

---

## 1. 起點：「藏鏡人那種端影音」
使用者問是否知道「藏鏡人的那種短影音」。

- 釐清「藏鏡人」：源自布袋戲《雲州大儒俠》的蒙面反派，躲在鏡屏後現身；引申為「藏在幕後操控的人」。
- 一度以為是「不露臉、變聲匿名爆料」的風格，但使用者澄清——

## 2. 釐清格式
使用者要的是：**鏡頭前有一個人（邊走邊講），鏡頭後有個看不見的人提問**。
→ 這就是「街訪 / 邊走邊聊 Walk & Talk / 跟拍訪談」，鏡頭後的提問者＝藏鏡人。

## 3. 確立目標與方向
- 用 **AI** 來做這種短影音。
- 為一個**新的社群帳號**用，所有影片都要在「類似的大方向」底下 → 主題要能**長期延續**。
- 偏**知識類**，例如「找工作 / 職場」。
- 終極目的：**內容導流 → 養紅帳號 → 開課程變現**。

收斂出的公式：**帳號大方向 = 固定的「提問主軸」× 無限的「受訪對象池」**。

## 4. 關鍵決策（透過提問確認）
| 項目 | 決定 |
|---|---|
| 受眾利基 | 先廣泛衝量，之後依數據 niche down |
| 課程主講 | 純虛擬品牌（不露真人臉） |
| AI 角色 | 混合：虛擬導師講觀念 + 情境模擬短劇 |
| 平台 | 先 IG Reels |

人設方向：建議「毒舌但中肯的職場學長/姐」。

## 5. 工具與自動化需求
使用者要的最終形態：
> 有一個**入口**，每天輸入想要的主題或調整方向 → 接下來**產腳本到產出影片都自動** → 有個地方可以**看成果** → 後續上傳手動就好。

確認：
- 入口形式 = **本機網頁 App**
- 影片引擎 = HeyGen 虛擬主播
- 預算 = **先用免費的**，後續可換工具

## 6. 重要查證發現
派研究 agent 查證後（2025–2026 最新）：
- ⚠️ **HeyGen 已於 2026/02 取消免費 API**，免費網頁版（每月 3 支、有浮水印）不能自動化。
- 其他環節都有好的免費方案：腳本 = Groq 免費 / Claude；配音 = HeyGen 內建 / edge-tts；字幕 = 本機 Whisper + FFmpeg。
- 結論：「完全免費 + 全自動 + 一致中文 avatar」目前幾乎不存在。

關於「我有 Claude 有用嗎」：
- ✅ Claude 對「腳本」這棒很有用（繁中、語氣最好）。
- ❌ Claude 不能生成 avatar 影片（那是另一種 AI）。
- 要讓 App 自動呼叫 Claude 需 **Anthropic API 金鑰**（月費訂閱不能直接給程式用）。

## 7. avatar 這棒的決定
三條路：HeyGen 付費（$0.5/支、全自動）/ **$0 半手動** / 開源免費（需 GPU）。
→ 使用者選 **「先 $0，avatar 半手動」**：其餘自動，唯獨 avatar 那步在 HeyGen 免費網頁手動做。

## 8. 最終方案（已實作）
> 本機網頁輸入主題 → App 自動產腳本 → 給「貼到 HeyGen 的逐字稿」→ 你做好 avatar mp4 上傳回來 → App 自動上字幕 + 轉 9:16 + 疊開場問題字卡 → 成果畫廊看片下載 → 手動發 IG。

- 技術棧：Python + FastAPI、Groq（腳本，可換 Claude）、faster-whisper + FFmpeg（字幕/合成）。
- 設計原則：**每個環節都是可抽換 provider**，日後升級付費全自動只改設定。
- 雙聲處理：導師 = avatar 口播；藏鏡人問題 = 開場字卡疊畫面（不需第二配音）。

## 9. 建置與驗證結果
已驗證：
- 全部 Python 語法、核心套件安裝、伺服器啟動 ✅
- `/api/config`、首頁 UI、建立/查詢 job ✅
- 無金鑰時優雅報錯 ✅
- 中文字型自動偵測（此 Mac 用 STHeiti / Heiti TC，因為沒有 PingFang）✅

待使用者完成的 pilot 前置：
1. `brew install ffmpeg`（此 Mac 尚未安裝）
2. 填免費 `GROQ_API_KEY` 到 `.env`
3. `pip install -r requirements.txt`（含 faster-whisper）
4. `uvicorn app.main:app` → http://localhost:8000

尚未驗證（pilot 第一支會驗到）：實際腳本生成（需金鑰）、影片合成（需 ffmpeg + 真的 HeyGen mp4）。

## 10. 後續進展（2026-06-29 同日延伸）

實際把工具跑起來、並重新評估 avatar 工具後，方向有幾項更新：

**環境前置（實測修正，原文件不夠）**
- Python 要用 **3.12**：系統的 3.14 太新，`onnxruntime`/`av`（faster-whisper 依賴）沒有對應套件，裝不起來。
- ffmpeg 要用**含字幕功能**的版本：homebrew 預設 `brew install ffmpeg` 沒編 libass（字幕）/libfreetype（字卡），改用 `brew install homebrew-ffmpeg/ffmpeg/ffmpeg`。
- 已實測整條媒體鏈通過（Groq 腳本 → 上傳 → Whisper 字幕 → FFmpeg 合成 9:16＋中文字幕＋開場字卡 → 下載）。

**工具介面/功能調整**
- 主題、內容線改為**下拉選單，且可留空 → AI 自動出題**（什麼都不選也能一鍵產腳本）。
- 腳本卡新增 **📋 複製完整腳本**（含開場字卡、畫面建議），與原本「只口播給 avatar 用」並存。
- 加入免費 **edge-tts**（台灣中文語音），可把口播逐字稿轉成語音檔。

**avatar 工具改道（重點）**
- HeyGen 受阻：免費版每支有浮水印、下載受限 → 棄用。
- 評估免費替代（Hedra 等）幾乎都有浮水印；**Sora 已於 2026/04 下架**，且本就不適合「照稿講話」。
- **定案改用 Kling 對口型**（朋友有帳號）：給「角色圖＋口播語音」讓虛擬人開口，支援中文、可鎖定同一張臉、能做走動運鏡；先手動，未來可接 **Kling API** 全自動。
- 角色圖（虛擬人的臉）用 **ChatGPT 生圖**（Claude 不能生圖），做一次固定使用。
